8.4.6　expand

RxJS还提供一个叫expand的操作符，这个操作符类似于mergeMap，但是，所有expand传递给下游的数据，同时也会传递给自己，就像是逐层“展开”所有的数据。

很明显，expand构成了一个逻辑死循环，除非发生异常，或者expand的函数参数project代码中限定在某些情况下不再产生数据，否则expand产生的绝对是一个永远不会完结的Observable对象，而且产生的数据量也很容易产生爆炸性增长。

下面代码通过expand产生2的N次方序列：

import {Observable} from 'rxjs/Observable';
import 'rxjs/add/observable/interval';
import 'rxjs/add/observable/of';
import 'rxjs/add/operator/delay';
import 'rxjs/add/operator/expand';

const source$ = Observable.of(1);
const result$ = source$.expand(
  (value, index) => Observable.of(value * 2).delay(10)
);

看看result$的结果很快就产生天文数字，就可以体会expand造成的“爆炸性”数据增长。
